# Bulk-Migration 実装順序（推奨）

## フェーズごとの具体的な実装ステップ

### 1. 開発基盤の整備
- 仮想環境・依存パッケージのセットアップ
- .env, config.json, ディレクトリ構成の整備
- logger.py（ログ出力の仕組み）雛形作成

### 2. Graph API認証基盤
- auth.py：MSALによる認証・トークン取得処理
- main.py：認証テスト用の最小スクリプト
- 認証情報の.env管理・config連携

### 3. 転送ロジックの基礎
- transfer.py：OneDrive/SharePointのディレクトリ再帰取得
- ファイル一覧取得・スキップリストの雛形
- 転送先ディレクトリ作成処理

### 4. ファイル転送処理
- OneDriveからダウンロードURL取得
- SharePointへPUTリクエストでアップロード
- チャンク転送・並列処理の実装
- 転送状況のログ化

### 5. 異常時対応・堅牢化
- 途中再開・リトライ機構
- ログ破損時の復旧処理
- ファイル名規則差異への対応
- 標準出力・ログ粒度の調整

### 6. 整合性・削除・レポート
- ハッシュ値比較による整合性チェック
- 転送元削除・削除ログ
- チェックサムレポート生成

### 7. テスト・運用性強化
- tests/配下にユニットテスト追加
- セットアップスクリプトの整備
- ドキュメントの充実

---

## 補足
- 各ステップごとにテスト・動作確認を推奨
- 必要に応じて順序や内容は柔軟に調整
- 実装の進捗や課題はdoc/配下に記録
