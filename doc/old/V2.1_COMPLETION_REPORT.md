## 🎯 v2.1の価値提案

### 🧹 核心機能: インテリジェントログクリーンナップ

```
問題: 設定変更後に古いログが残ると...
❌ 重複チェックの誤作動
❌ 統計情報の混在  
❌ デバッグ時の混乱

解決: v2.1の自動ログクリーンナップ
✅ 設定変更を自動検出
✅ ユーザーフレンドリーな確認プロンプト
✅ 安全な自動バックアップ
✅ 新規実行での整合性確保
```

### 💡 ユーザーエクスペリエンス

**従来 (v2.0)**:
- 設定変更後、手動でログ削除が必要
- 古いログとの混在リスク
- 運用者の判断に依存

**v2.1**:
```bash
🔍 設定変更が検出されました:
📝 OneDrive移行元パス: Music → Photos
📝 SharePoint移行先パス: 音楽 → 写真

❓ ログファイルをクリーンナップしますか？
   [y] はい（推奨）- 古いログをバックアップして新規開始
   [n] いいえ - 既存ログに追記
   [v] 表示 - 既存ログの内容を確認

選択してください [y/n/v]: y
📁 ログを logs/backup/20250630_120000 にバックアップしました
🚀 ログクリーンナップ完了。新規実行を開始します。
```
